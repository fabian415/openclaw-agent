### 2. Sim-to-Real 資料轉換問題
- **問題描述**：從 Simulator（**iSAAC**）輸出 **HDF5**，再轉換為 **LeRobot** Dataset，訓練出的模型在 Real 環境表現不佳
- **調查過程**：
  - 懷疑是 MP4 **Codec** 差異：**Physical AI Server** 使用 **H.264**（內建 FFmpeg），HDF 轉換用 **LeRobot 內建 AC0**
  - 修改 Codec 後測試，結果相同 → Codec **非主因**
  - 確認 LeRobot 版本差異：Physical AI Server 用 `0.2.0/0.2.2`，LeRobot 現版為 `0.3.1`
- **暫行方案**：直接以 **Physical AI Server** 錄製資料（Simulator Camera → Physical AI Server 錄製），確認可行
- **後續行動**：統一 Library 版本再測一次，找出 HDF→Dataset 轉換的根本問題

### 4. AI 設備助手（Smart Device Assistant）
- **目標功能**：
  - 定期從 Server API 取得設備資料，AI 分析後產出**日報 / 週報**
  - 預警項目：儲存空間預測、CPU/NPU 使用率異常、多次登入失敗、Event Log 分析
  - 提供**自動修復建議**，甚至授權 AI 執行清理動作
- **現況**：目前日報表功能開發中（資料範圍仍在確認）
- 使用模型：**GPT-5 mini**（免費，zero cost request）
- 特色：**長期記憶** — 隨使用時間增長，AI 越了解操作者習性

---

### 5. 個人 AI 助手（RAG / Memory）
- 架構：**LlamaIndex** + 向量資料庫（VectorDB）+ 本地或雲端模型
- 流程：讀取 PDF / 檔案 → AI 解析結構化資料 → 存入向量庫 → 使用者提問時撈出相關資料再問答
- 優點：減少 Context Window 用量，長期記憶具個人化
- 限制：新 Session 啟動後記憶需重載（設計中）
- 進度：PDF 讀取 + 存庫流程完成，Skill 自動化流程仍在調整

### 7. QA 測試 / Memory Leak 問題（SOR）
- **問題**：System Store 2 YouTube OC issue，overnight 測試後系統 Reset
- **根因**：推測為 **DSS 框架 Memory Leak**（記憶體持續成長）
- **暫行方案**：每兩小時重啟 DSS，確保電流測試可持續 48 小時
- **狀態**：電流目前未頂到上限 ✅
