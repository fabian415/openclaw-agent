# OpenClaw 會議記錄 Skill 專案洞察

> 最後更新：2026-04-10
> 對外發表成熟度：Soft Launch Ready
> 專案別名：會議記錄 Skill、Meeting Transcription Workflow、會議助理、PM 助理候選能力

## 一頁摘要
- **目前階段**：可用原型，開始產品化
- **核心價值主張**：讓錄音、逐字稿、會議整理、專有名詞補強、語者對齊與寄送形成一個可執行的會議工作流。
- **本次更新重點**：
  - 工作流架構與技術拆解更完整
  - 語者對齊、專有名詞補強、metadata、模板化輸出都已納入
  - 團隊開始要求它往 PM 助理，而不只是 meeting minutes tool 發展
  - ready-to-send 人工校對節點被明確提出
- **PM 判斷**：這個東西已經不只是內部玩具，是真的有產品味；但若要正式對外，正確性與 initial setup 仍要補齊。

## 專案目標
- 把會議錄音變成可用的逐字稿、會議記錄、決議事項與行動項目。 `來源：2026-03-20 逐字稿；2026-04-10 逐字稿`
- 進一步沉澱跨次會議知識，朝 PM 助理、產品特色整理、產品頁 / PPT 輔助產出發展。 `來源：2026-04-10 逐字稿；2026-04-10 會議記錄`

## 欲解決的痛點
- 單純 ASR 無法處理語者身份、專有名詞、專案脈絡與會議後續行動。 `來源：2026-04-10 逐字稿`
- 逐字稿很長，若缺少結構化與模板整理，無法真正被 PM / RD / 管理者使用。 `來源：2026-04-10 逐字稿`
- 完全自動寄出風險太高，需要人工校對節點。 `來源：2026-04-10 逐字稿；2026-04-10 會議記錄`

## 目前進展
### 工作流成熟度
- 已具備錄音 → metadata → WhisperX 轉錄 → 整理模板 → email 輸出的完整流程。 `來源：2026-04-10 逐字稿`
- 支援商務會議、演講、訪談等不同模板方向。 `來源：2026-04-10 逐字稿`

### 技術加值能力
- 語者對齊：透過個人聲音樣本 10–30 秒做 alignment。 `來源：2026-04-10 逐字稿`
- 專有名詞補強：先從文件抽詞，再灌入 ASR prompt / keyword prompt。 `來源：2026-04-10 逐字稿`
- metadata：會議時間、主題等會一起送進流程。 `來源：2026-04-10 逐字稿`

### 現有限制
- 本地轉錄速度仍偏慢，1.5 小時音檔可能要約 1 小時處理。 `來源：2026-04-10 逐字稿`
- 專有名詞太多時，prompt 過長會影響 inference。 `來源：2026-04-10 逐字稿`
- 有些內容會遺失或轉譯不準，尤其是專業詞與中英夾雜。 `來源：2026-04-10 逐字稿`
- 整理結果雖然完整，但仍受整理模型品質影響。 `來源：2026-04-10 逐字稿`

### 角色升級：從會議記錄到 PM 助理
- Louis 明確提出，理想狀態不只是產出會議紀錄，而是要能持續追蹤同一專案多次會議、抽出產品特色、協助生成產品頁、PPT，並提醒 next action。 `來源：2026-04-10 逐字稿；2026-04-10 會議記錄`
- 這使它從 meeting minutes tool，升級為 PM workflow 的入口能力。 `來源：2026-04-10 逐字稿推定`

- **目前判斷依據**：工作流完整度高，但準確率、速度、人工校對與 setup 還需補齊。 `來源：2026-04-10 逐字稿`
- **距離成熟對外版的缺口**：
  - ready-to-send 人工複核步驟
  - initial setup 文件化（個人語音樣本、術語、模型選型）
  - 專有名詞詞庫與語者資料管理
  - 多次會議串接到專案知識庫能力
- **預估時程**：若聚焦內部 / 特定團隊使用，已接近可正式導入；若要作為產品對外賣，還需要一輪流程產品化。 `來源：2026-04-10 逐字稿；2026-04-10 會議記錄`

## 技術亮點與重大突破
### 技術亮點
- 已形成從 **錄音、metadata、WhisperX 轉錄、語者對齊、專有名詞補強、模板整理到 email 發送** 的完整工作流。 `來源：2026-04-10 逐字稿`
- 支援商務會議、演講、訪談等不同模板方向，可依情境輸出不同格式。 `來源：2026-04-10 逐字稿`
- 透過個人聲音樣本做 **speaker alignment**，不只分出 speaker 1/2/3，還朝向對應真實講者身份推進。 `來源：2026-04-10 逐字稿`
- 可先從文件抽取專有名詞，再灌入 ASR keyword prompt，提升專業術語辨識品質。 `來源：2026-04-10 逐字稿`

### 重大突破
- 這套流程已從單純的逐字稿工具，升級成具備 **會議知識結構化能力** 的工作流原型。 `來源：2026-04-10 逐字稿`
- 團隊已明確提出要把它往 **PM 助理、產品特色抽取、產品頁與 PPT 素材生成** 發展，這是產品定位上的重要突破。 `來源：2026-04-10 逐字稿；2026-04-10 會議記錄`

- 「這不只是會議摘要工具，而是把會議內容轉成可被 PM、主管與團隊持續使用的知識資產。」

## 技術實踐與落地方式

- 技術組件包含 OpenClaw、WhisperX、本地 GPU 轉錄、術語抽取、語者對齊、模板化整理與 email 發送。 `來源：2026-04-10 逐字稿`
- 會議開始前可上傳個人語音樣本與專業術語，作為 initial setup，改善後續 speaker 與專有名詞辨識品質。 `來源：2026-04-10 逐字稿`

### 驗證方式
- 團隊以真實雙週會 / 技術會議作為驗證場景，確認整套流程可產出逐字稿、決議事項、代辦事項與摘要。 `來源：2026-03-20 逐字稿；2026-04-10 逐字稿`
- 目前也已驗證不同模板與本地 / 雲端模型的效果差異。 `來源：2026-04-10 逐字稿`

### 已知限制
- 本地轉錄速度偏慢；長音檔處理時間仍高。 `來源：2026-04-10 逐字稿`
- 術語太多時，prompt 過長會影響 inference。 `來源：2026-04-10 逐字稿`
- 整理正確率仍需人工複核，不能直接無審核寄出。 `來源：2026-04-10 逐字稿；2026-04-10 會議記錄`
