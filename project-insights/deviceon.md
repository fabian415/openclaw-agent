# DeviceOn 專案洞察

> 最後更新：2026-04-10
> 對外發表成熟度：Soft Launch Ready
> 專案別名：DeviceOn Server、DeviceOn AI Assistant、智慧維運助手

## 一頁摘要
- **目前階段**：功能擴充與智慧化驗證中
- **核心價值主張**：從傳統裝置監控進一步升級為具備 AI 分析、風險預警、記憶與建議能力的智慧運維助手。
- **本次更新重點**：
  - AI 助手從概念規劃進一步進入實作驗證
  - 已可產出 server 狀態、設備數、事件統計、憑證與空間資訊
  - memory 穩定性仍是主要阻塞
  - Jetson Device Agent ID / MAC 誤判問題已找到修正方向
- **PM 判斷**：已具備相對具體的應用場景與客戶價值，若把 use case 講清楚，已接近可對外做定向溝通。

## 專案目標
- 讓 DeviceOn 從 dashboard 型管理工具，進化為具備分析、提醒、建議與記憶能力的智慧助手。 `來源：2026-03-20 逐字稿；2026-03-20 會議記錄`
- 協助管理者快速理解設備狀態、風險、憑證、儲存空間、日誌與 CVE 問題。 `來源：2026-03-20 逐字稿；2026-03-20 會議記錄`

## 欲解決的痛點
- 現有 dashboard 只能顯示狀態，無法主動分析潛在問題。 `來源：2026-03-20 逐字稿`
- 管理者真正關心的是少數幾個高風險重點，而不是大量原始數據。 `來源：2026-03-20 逐字稿`
- 裝置端與 server 端的事件、日誌、CVE 與容量風險需要更智慧化的整合分析。 `來源：2026-03-20 逐字稿；2026-03-20 會議記錄`

## 目前進展
### AI 智慧助手
- 已規劃從 Server 定期抓資料，產生日報 / 週報，涵蓋設備上下線、憑證到期、CPU / Memory 異常、登入失敗、Storage 預測與 CVE 比對。 `來源：2026-03-20 逐字稿；2026-03-20 會議記錄`
- 到 2026-04-10 時，已可拉 server 狀態、剩餘空間、GPU 使用量、憑證剩餘天數、設備總數、連線狀態與事件統計。 `來源：2026-04-10 逐字稿`
- 最大阻塞仍是 memory 回收後上下文掉失，導致技能執行不穩定。 `來源：2026-04-10 逐字稿`

### 客戶與市場進展
- Ponseca（羅馬尼亞）已對 DeviceOn 功能驗證表示滿意，正進一步在 production environment 驗證，若順利有機會下單。 `來源：2026-03-20 逐字稿；2026-03-20 會議記錄`
- Swisscon 對 AGI documentation 與 Windows + WSL 場景下的控制能力有需求，但細節尚待釐清。 `來源：2026-03-20 逐字稿；2026-03-20 會議記錄`

### 版本與功能更新
- DeviceOn 2.0.33 已更新 Vigio / Vigil VNC 版本與 security 相關頁面。 `來源：2026-03-20 逐字稿；2026-03-20 會議記錄`
- CPU code usage 監控基礎功能可行，待整合。 `來源：2026-03-20 逐字稿；2026-03-20 會議記錄`
- ARK 353 HPAC 問題已確認主因為 EC 進版與 DeviceOn Server 版本不同步。 `來源：2026-03-20 逐字稿；2026-03-20 會議記錄`
- Jetson 平台因虛擬網卡 `L4TBR` 被誤抓為最小 MAC，導致 Device Agent ID 問題；目前做法為忽略該虛擬卡。 `來源：2026-04-10 逐字稿；2026-04-10 會議記錄`

## 產品化敘事成熟度
- **目前狀態**：中高
- **已具備**：
  - 明確的智慧維運 use case
  - 週報 / 風險預警 / 憑證 / 空間 / 事件統計等典型場景
  - 歐洲客戶驗證訊號
- **仍欠缺**：
  - memory 穩定性
  - CVE 與剩餘空間預測等高階功能完整落地
  - 更清楚的管理者 persona 與示範報表包裝

## MP 狀態
- **是否已 MP**：主體產品已在市場運行；AI 助手能力尚未 MP
- **目前判斷依據**：既有 DeviceOn 主體產品已成熟，但新增的 AI 智慧助手仍在規劃與驗證中。 `來源：2026-03-20 逐字稿；2026-04-10 逐字稿`
- **距離成熟對外版的缺口**：
  - memory 與 session continuity
  - CVE compare、storage 預測、自動修復建議
  - 管理者導向的報表與互動設計
- **預估時程**：若先從 server 端週報表與異常摘要切入，短期內可形成可展示版本；正式對外包裝仍需更多客戶案例支持。 `來源：2026-03-20 逐字稿推定；2026-04-10 逐字稿`

## 技術亮點與重大突破
### 技術亮點
- 已能從 DeviceOn Server 拉取 **設備數、連線狀態、GPU 使用量、剩餘空間、憑證剩餘天數與事件統計**，具備智慧維運報表雛形。 `來源：2026-04-10 逐字稿`
- 規劃方向已不只停留在告警，而是往 **CVE 比對、容量預測、異常分析、建議行動** 推進。 `來源：2026-03-20 逐字稿；2026-03-20 會議記錄`
- 助手定位開始強調「有記憶」、「知道 manager 真正關心哪三件事」，具備從 dashboard 走向 AI 運維助理的潛力。 `來源：2026-03-20 逐字稿`

### 重大突破
- 已從概念規劃進入部分實作驗證，代表 DeviceOn AI Assistant 不再只是紙上概念。 `來源：2026-04-10 逐字稿`
- Jetson Device Agent ID / MAC 問題已找到具體修正方向，顯示團隊開始處理真實部署環境中的 platform-specific 問題。 `來源：2026-04-10 逐字稿；2026-04-10 會議記錄`

### 對外可用技術敘事

- 「相較傳統 dashboard 只顯示狀態，DeviceOn 正朝向具備記憶、趨勢判讀與風險預警能力的智慧維運助手發展。」
